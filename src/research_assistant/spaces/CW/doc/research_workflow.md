# Research Workflow Guide - CW (Case Writing)

> **Workflow**: `research`
> **Purpose**: Guide case research and data gathering strategy
> **Last Updated**: 2026-02-02

---

## Expected Input
- `task`: Research objective for case development

## Research Stages

### Stage 1: Topic Validation
- Industry/sector analysis
- Timeliness assessment
- Access feasibility check

### Stage 2: Source Mapping
- Primary sources (interviews, site visits)
- Secondary sources (reports, filings, news)
- Academic sources (theories, frameworks)

### Stage 3: Data Collection Plan
- Interview protocol development
- Document request list
- Timeline and milestones

---

---

## Grounding Rules

### MUST DO
- [ ] Map to case development stages
- [ ] Identify KB materials that inform research
- [ ] Consider data accessibility realistically
- [ ] Reference similar published cases for benchmarks

### MUST NOT DO
- [ ] Propose research without access consideration
- [ ] Ignore ethical/confidentiality constraints
- [ ] Skip teaching utility alignment
