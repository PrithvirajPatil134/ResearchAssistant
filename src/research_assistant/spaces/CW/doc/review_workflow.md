# Review Workflow Guide - CW (Case Writing)

> **Workflow**: `review`
> **Purpose**: Review case submissions against case writing standards
> **Last Updated**: 2026-02-02

---

## Expected Input
- `submission_path`: Path to case draft or teaching note for review

## Review Criteria (Prof. Kakoli Sen)

### Case Narrative Review
- [ ] Clear protagonist with decision authority
- [ ] Compelling opening hook
- [ ] Sufficient context without overwhelming data
- [ ] Time pressure/urgency established
- [ ] Decision point clearly framed
- [ ] Appropriate length (5-15 pages typical)

### Teaching Note Review
- [ ] Synopsis matches case narrative
- [ ] Learning objectives are measurable
- [ ] Discussion questions scaffold analysis
- [ ] Board plan supports flow
- [ ] Analysis frameworks appropriate
- [ ] Epilogue/resolution included if relevant

### Data & Exhibits Review
- [ ] Exhibits referenced in narrative
- [ ] Data supports decision analysis
- [ ] No confidential information exposed
- [ ] Sources cited appropriately

---

## Grounding Rules

### MUST DO
- [ ] Reference specific template criteria
- [ ] Cite examples from published cases (C1-C15)
- [ ] Provide actionable recommendations
- [ ] Maintain constructive tone

### MUST NOT DO
- [ ] Give generic feedback without specifics
- [ ] Skip teaching utility assessment
- [ ] Ignore narrative quality
