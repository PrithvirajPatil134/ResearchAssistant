# Review Workflow Guide - DBA

> **Workflow**: `review`
> **Purpose**: Review thesis proposal submissions against doctoral standards
> **Last Updated**: 2026-01-31

---

## Workflow Steps
```
1. ReaderAgent → Scans submission + KB standards
2. ThinkingModule → Evaluates against criteria
3. AnalystAgent → Scores review quality
4. ReviewerAgent → Validates feedback
5. OutputWriter → Saves review to /output
```

## Expected Input
- `submission_path`: Path to student's proposal section/draft

## Expected Output
- Structured review with strengths, areas for development, recommendations
- Grounded in DBA Thesis Proposal Template standards
- Actionable feedback for improvement

---

## Grounding Rules

### MUST DO
- [ ] Evaluate against template section requirements
- [ ] Cite specific criteria from Prof. Cardasso's guidance
- [ ] Provide constructive, actionable feedback
- [ ] Reference academic writing standards

### MUST NOT DO
- [ ] Give vague feedback without specific references
- [ ] Be overly critical without suggestions
- [ ] Ignore methodological rigor requirements
